Initial Configuration

You can use your keyboard to move between slides. 

When you start Threats Manager Platform,

the Consent Required dialog is shown...

... please read it carefully and if you agree click on the Yes button.

This opens the tool, which is called Threats Manager Studio.

Before using it, we need to configure some parameters.

Click on the menu File.

And then click Options.

Options contains three pages.

The Extensions page allows specifying the folder where the Extensions are installed.

Normally, you would click Add Default to add the standard path.

You can also specify another path, if you want.

Then you can specify the prefix for the Assemblies containing Extensions.

This is only for performance purposes, but is highly recommended.

To do that, you need first to enable the related checkbox...

... and then to specify the prefix. Normally you should use Add Default.

The last step for this page is to enforce certificate checking.

This is highly recommended for security purposes, to avoid loading untrusted Extensions.

To do that, first enable the checkbox.

And then click Add Default.

Now we can switch to the Smart Save page.

Smart Save is a powerful feature to avoid losing your work. It saves the Threat Model automatically, maintaining a number of old versions.

To configure it, please enable Smart Save first.

Then you have to configure the number of saves to maintain (1) and how frequently you want the tool to automatically save your models (2).

2

Now we can move to the Other Settings

Here you can set the User Dictionary.

Normally you would not need to configure it.

When done, please click Ok and the close the Threats Manager Studio.

The next time you'll open it again, you'll see your additional Extensions.

Thank you for watching!
